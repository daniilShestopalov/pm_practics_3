### Анализ потребности в сотрудниках для проекта "Travel Diary App"
## Основа для расчета
# Исходные данные оценки

- Трудозатраты: 2,948 человеко-часов (16.75 человеко-месяцев)
- Use Case Points: 105.3 UCP
- Сложность: Средняя (105 UCP) 

## Cостав команды
# Базовый состав (минимальный вариант)

Должность	Количество	Фокус нагрузка	Обоснование

- Full-stack разработчик	2 чел	60% бэкенд, 40% фронтенд	Основная разработка, покрытие всех технологий
- Mobile разработчик	1 чел	100% мобильная разработка	Специализация на iOS/Android, нативные компоненты
- UI/UX дизайнер	1 чел	100% дизайн	Высокие требования к UI/UX (T3=5, T6=5)
- QA инженер	0.5 чел	100% тестирование	Частичная занятость, кросс-функциональное тестирование
- Project Manager	0.5 чел	100% управление	Частичная занятость, координация команды
  
**Итого:** 5 человеко-эквивалентов

# Оптимальный состав (рекомендуемый)

Должность	Количество	Специализация	Критичные навыки

- Backend разработчик	1 чел	Python/Django или Node.js	API, базы данных, интеграции
- Frontend разработчик	1 чел	React/TypeScript	Веб-интерфейсы, карты, UI
- iOS разработчик	1 чел	Swift, UIKit/SwiftUI	Нативная iOS разработка
- Android разработчик	1 чел	Kotlin, Jetpack Compose	Нативная Android разработка
- UI/UX дизайнер	1 чел	Figma, прототипирование	Mobile-first дизайн, UX исследования
- QA инженер	1 чел	Автотесты, ручное тестирование	Мобильное тестирование, кросс-платформенное
- DevOps инженер	0.5 чел	AWS/GCP, CI/CD	Инфраструктура, развертывание
- Project Manager	1 чел	Agile, Scrum	Управление, коммуникация
  
**Итого:** 7.5 человеко-эквивалентов

## Расчет сроков выполнения
# По методу Монте-Карло 

- Минимальный (5 чел)	4-6 месяцев	70% за 5 месяцев
- Оптимальный (7.5 чел)	3-4 месяца	85% за 3.5 месяца

# Факторы, влияющие на сроки (Event Chain Methodology)

- **Критические события:** Изменение требований к картографическому функционал
- **Цепочки рисков:** Задержка интеграции с внешними API → сдвиг сроков тестирования
- **Позитивные события:** Наличие готовых библиотек для работы с картами
  
## Специальные требования к компетенциям
# Технические специализации (на основе Use Case Groups)

**Группа 1: Создание и запись путешествия**

- **Требуемые навыки:** Геолокация, работа с камерой, медиа-обработка
- **Критичные технологии:** GPS API, EXIF данные, оптимизация изображений

**Группа 2: Организация и анализ**

- **Требуемые навыки:** Картографические сервисы, аналитика, поисковые системы
- **Критичные технологии:** MapKit/Google Maps, алгоритмы поиска, базы данных

**Группа 3: Социальное взаимодействие**

- **Требуемые навыки:** Социальные функции, генерация PDF, безопасность
- **Критичные технологии:** OAuth, PDF generation, REST API security

# Уровень опыта требований

- Mobile разработчики	Middle+	Опыт с геолокацией, камерой, оффлайн-режимом
- Backend разработчик	Middle+	Опыт с высоконагруженными API, интеграциями
- UI/UX дизайнер	Middle+	Портфолио мобильных приложений, юзабилити-тесты
- QA инженер	Middle	Опыт мобильного тестирования, автоматизация
  
## Распределение работы (Feature Architectural Similarity)
# Группировка функциональности по командам

**Команда A: Мобильная платформа (3 чел)**

- iOS/Android разработчики + Mobile QA
- Use Cases: 1-8 (все мобильные функции)

**Команда B: Бэкенд и веб (2.5 чел)**

- Backend + Frontend + DevOps
- Use Cases: 1, 4, 7, 8 (API, аналитика, поиск)

**Команда C: Дизайн и управление (2 чел)**

- UI/UX + Project Manager
- Сквозная поддержка всех Use Cases

# План распределения нагрузки

- Месяц 1-2: Ядро приложения (Use Cases 1-4)
- Месяц 2-3: Расширенная функциональность (Use Cases 5-8)  
- Месяц 3-4: Социальные функции и полировка (Use Cases группы 3)

# Заключение

- Рекомендуемый состав команды: 7-8 специалистов
- Срок реализации: 3-4 месяца с вероятностью успеха 85%
- Критичные специализации: mobile разработка, UI/UX дизайн
- Ключевые технологические стеки: нативные mobile технологии + современный backend
